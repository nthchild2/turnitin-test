# Execution Rules

When writing or modifying source code:

- Follow the plan step by step. Do not skip or reorder steps.
- Write minimal changes. Do not refactor unrelated code.
- Preserve existing patterns: naming, imports, formatting, file structure.
- Add all necessary imports and dependencies.
- If a step is blocked or unclear, stop and report instead of improvising.
- Do not re-scope. If the plan seems wrong, flag it — do not silently fix it.
